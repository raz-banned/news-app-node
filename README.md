# News App Server

Мини-сервер для новостного приложения, построенный на базе Node.js, Express, TypeScript и SQLite. Проект предоставляет полноценный REST API для управления статьями (новостями) с механизмом регистрации пользователей и аутентификацией на основе JSON Web Tokens (JWT).

## 🚀 Стек технологий

- **Runtime:** Node.js
- **Язык:** TypeScript
- **Фреймворк:** Express 5
- **База данных:** SQLite (через драйвер `better-sqlite3`)
- **Аутентификация:** JWT (`jsonwebtoken`) & хэширование паролей (`bcryptjs`)
- **Пакетный менеджер:** pnpm
- **Деплой / Окружение:** Настройки CORS для Vercel

## 🛠️ Установка и запуск

### 1. Клонирование репозитория и установка зависимостей

Убедитесь, что у вас установлен `pnpm` версии 10+.

```bash
pnpm install
```

### 2. Настройка окружения

Создайте файл .env в корневой директории проекта и добавьте туда секретный ключ для генерации JWT и порт (опционально):

```env
PORT=3000
JWT_SECRET=your_super_secret_key_here
```

### 3. Заполнение базы данных (Seed)

Для автоматического создания таблиц и наполнения базы данных тестовыми пользователями и статьями выполните:

```bash
pnpm run seed
```

Это очистит старые данные и создаст администратора admin с паролем password123, а также три стартовые новости.

### 4. Запуск приложения

Режим разработки (с hot-reload):

```bash
pnpm run dev

```

Сборка и запуск в production:

```bash
pnpm run build
pnpm start
```

## 📂 Структура проекта

- src/db/ — Инициализация SQLite, схемы таблиц и скрипт наполнения (seed).
- src/middleware/ — Посредники для проверки авторизации (JWT-валидация).
- src/routes/ — Логика эндпоинтов API (auth и articles).
- src/index.ts — Точка входа приложения, настройка CORS и промежуточного ПО.

## 📡 Эндпоинты API (API Reference)

### Аутентификация (/api/auth)

- POST /register — Регистрация нового пользователя (требуется имя, email и пароль).
- POST /login — Вход в систему (требуется email и пароль).

### Статьи (/api/articles)

- GET / — Получить все статьи (требуется авторизация).
- POST / — Создать новую статью (требуется авторизация, требуется title и
  content).
- GET /:id — Получить статью по ID (требуется авторизация).
- PUT /:id — Обновить статью по ID (требуется авторизация, может обновлять title и content).
- DELETE /:id — Удалить статью по ID (требуется авторизация).

## 🔐 Безопасность

- Пароли пользователей хэшируются с помощью bcryptjs перед сохранением в базу данных.

- Доступ к роутам модификации данных ограничен middleware-функцией requireAuth.

- Внедрена базовая настройка CORS, разрешающая запросы со всех источников (\*), что упрощает локальную разработку frontend-части.
